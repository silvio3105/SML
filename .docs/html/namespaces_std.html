Scan C-string for wanted token.

This function is replacment for scanf function. C-string format must be known to use this function. Function returns SSTD_OK and result even if \0 is encountered during searching for token's end. Set \0 as first separator to indicate that wanted token starts from begining of input C-string.

Example #1: C-string test1-test2.hello,test123 and call sscan(string,'.',0,'\0',0,output) -> output will be pointing at letter h and length will be 13. Output C-string will be hello,test123

Example #2: C-string test1,test2-test3-test4.test5,test6 and call sscan(string,'-',1,',',0,output) -> output C-string will be test4.test5 length will be 11. 1 because there is 1 - before wanted token, dash between test2 and test3 0 because there is 0 , after sepBegin

Example #3: C-string test1,test2,test3-test4.test5,test6 and call sscan(string,'\0',0,',',1,output) -> output C-string will be test1,test2 length will be 11. \0 as begin separator indicates that wanted token starts from begining. In that case, sepCntBegin parameter makes no effect.

See example code for more info.

Parameters
inputPointer to first character in C-string.
sepBeginCharacter used as separator before wanted token.
sepCntBeginNumber of separators before wanted token.
sepEndCharacter used as separator after wanted token.
sepCntEndNumber of separators after wanted token is found.
outputReference to char output data.
modifySet to 1 to replace found separator with \0 char.
Returns
SSTD_NOK if no token was found.
SSTD_OK if token was found.
Warning
sepCntBegin and sepCntEnd start from 0!
Note
C-string has to be NULL terminated.
Function scans from left to right.

Scan C-string for wanted tokens.

This function works same as uint8_t sscan(char* input, char separator, uint8_t separatorCnt, sStd::data<char>& output) Only difference is that this function can scan for multiple tokens. If output result is nullptr then wanted token was not found. See example code for more info.

Parameters
inputPointer to first character in C-string.
dataPointer to input-output data list.
lenLength of data
sortedSet to 1 if input list is sorted(from left to right). Function will continue where it ended during scan for previous entry in input list. This parameter is optional.
modifySet to 1 to replace found separator with \0 char.
Returns
Number of found tokens.
Warning
modify can be used only when data list is sorted
Note
If data list is sorted, set param sorted to 1 to make scanning faster.
C-string has to be NULL terminated.
Function scans from left to right.
