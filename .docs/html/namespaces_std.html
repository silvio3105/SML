Scan C-string for wanted parameter.

This function is replacment for scanf function. C-string format must be known to use this function. Function does not modify C-string. Example #1 C-string test1-test2.hello,test123 and call sscan(string,'.',0,'\0',0,output) output will be pointing at letter 'h' and length will be 13. Output C-string will be hello,test123

Example #2 C-string test1,test2-test3-test4.test5,test6 and call sscan(string,'-',1,',',0,output) output C-string will be test4.test5 length will be 11. 1 because there is 1 - before wanted parameter, dash between test2 and test3 0 because there is 0 , after first separator is found(dash). See example code for more info.

Parameters
inputPointer to first character in C-string.
sepBeginCharacter used as separator before wanted parameter.
sepCntBeginNumber of separators before wanted token.
sepEndCharacter used as separator after wanted parameter.
sepCntEndNumber of separators after wanted parameter is found.
outputReference to char output data.
Returns
SSTD_NOK if no token was found.
SSTD_OK if token was found.
Warning
This function works only with char types!
Note
C-string has to be NULL terminated.
Function scans from left to right.

uint8_t sStd::sscan ( char *  input,
scanData &  data 
)

Scan C-string for wanted parameters.

This function works same as uint8_t sscan(char* input, char separator, uint8_t separatorCnt, sStd::data<char>& output) Only difference is that this function can scan for multiple parameters. If output result is nullptr then wanted parameter was not found. See example code for more info.

Parameters
inputPointer to first character in C-string.
dataReference to input-output data list.
Returns
SSTD_NOK if no tokens were found.
Number of found tokens.
Warning
This function works only with char types!
Note
data list does not have to be sorted since this function depends on uint8_t sscan(char* input, char separator, uint8_t separatorCnt, sStd::data<char>& output) and it will start from beginning for every entry in data
C-string has to be NULL terminated.
Function scans from left to right.
