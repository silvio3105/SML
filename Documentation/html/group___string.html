Generic C-string parser.

This function is replacment for scanf function. C-string format must be known to use this function. Function returns SML::Return_t::Ok and result even if \0 is encountered during searching for token's end. Set \0 as start separator to indicate that wanted token starts from begining of input C-string.

Example #1: C-string test1-test2.hello,test123 and call parse(string,'.',0,'\0',0,output) -> output will be pointing at letter h and length will be 13. Output C-string will be hello,test123

Example #2: C-string test1,test2-test3-test4.test5,test6 and call parse(string,'-',1,',',0,output) -> output C-string will be test4.test5 length will be 11. 1 because there is 1 - before wanted token, dash between test2 and test3 0 because there is 0 , after startSeparator

Example #3: C-string test1,test2,test3-test4.test5,test6 and call parse(string,'\0',0,',',1,output) -> output C-string will be test1,test2 length will be 11. \0 as begin separator indicates that wanted token starts from begining. In that case, startSeparatorCnt parameter makes no effect.

See example code for more info.

Parameters
inputPointer to first character in C-string.
startSeparatorCharacter used as separator before wanted token.
startSeparatorCntNumber of separators before wanted token.
endSeparatorCharacter used as separator after wanted token.
endSeparatorCntNumber of separators after wanted token is found.
outputReference to char output SML::Data
modifySet to SML::Answer_t::Yes to replace found separator with \0 char. Not required.
Returns
SML::Return_t::Nok if no token was found.
SML::Return_t::Ok if token was found.
Warning
startSeparatorCnt and endSeparatorCnt start from 0!
Note
C-string has to be NULL terminated.
Function parses from left to right.

Parse C-string for multiple strings.

This function works same as SML::parse Only difference is this function parse for multiple strings. If output result is nullptr then wanted part was not found. See example code for more info.

Parameters

listPointer to parser list.
lenLength of list
modifySet to SML::Answer_t::Yes to replace found separator with \0 char. Not required.
sortedSet to SML::Answer_t::Yes if input list is sorted(from left to right). Function will continue where it stopped during parse for previous entry in input list. Not required.
Returns
Number of parsed strings.
Warning
modify can be used only when list is sorted
